# Pedagogy Skill — Russian A1-to-A2 Teaching Knowledge for Game Context

Apply the following principles whenever you create, modify, or review NPC dialogue, mission definitions, vocabulary entries, or tutor prompts.

## Standing knowledge — game-specific

- Vocabulary is introduced contextually through NPC dialogue and mission requirements — never through explicit vocabulary lists
- Grammar is encountered through doing, not through explanation — the player uses accusative before they know what it is
- The A1-to-A2 sequence maps to location unlock order exactly: Apartment/Park = A1, Cafe/Market = A1+, Station/Police = A2
- Every word introduced in location N must appear again naturally in location N+1 — no word is introduced once and dropped
- Russian text rules: minimum 15px, always show translation below, always show transliteration for new words on first encounter

## A1-to-A2 Russian sequencing principles

- **Comprehensible input first**: New grammar or vocabulary is always introduced through NPC dialogue the learner can mostly understand. Never lead with an abstract rule — show it in a conversation, then name it.
- **Pattern before exception**: Teach the regular pattern until it is automatic, then introduce exceptions. At A1, only the most common exceptions matter.
- **Spaced exposure over massed practice**: Introduce a word through an NPC, revisit it in the next location's dialogue, then test it after a gap.
- **Receptive before productive**: Learners should recognise before they produce. Multiple-choice dialogue options precede text-input requirements.
- **One new difficulty at a time**: If introducing a new case ending, use familiar vocabulary. If introducing new vocabulary, use a familiar grammatical structure.
- **Concrete before abstract**: Teach "на столе" as something an NPC says naturally before explaining the prepositional case as a system.

## Common English speaker errors in Russian at A1

These are the mistakes the player is most likely to make. Mission design should target these:

1. **Case endings**: English has no case system. Dialogue choices should include plausible wrong-case options.
2. **Verb aspect confusion**: At A1, focus only on present tense imperfective. Mention perfective existence but don't test it until Station/Police.
3. **Gender agreement**: Adjective-noun gender agreement is unfamiliar. NPCs should model correct agreement and react to wrong agreement.
4. **Stress placement**: Russian stress is unpredictable. All vocabulary entries mark stress.
5. **Word order flexibility**: Introduce gently through NPC speech patterns. Don't test on it at A1.

## What works in game-based language learning

- **Contextual vocabulary**: Words learned from NPCs in meaningful situations, not isolated lists.
- **Immediate NPC feedback**: Wrong answer gets confused NPC reaction + one-sentence explanation, not "incorrect."
- **Low-stakes missions**: Missions are for learning. Only chapter tests are scored.
- **Graduated difficulty within locations**: First missions are recognition (choose the right option), later missions are production (type the right phrase).

## What fails — avoid these patterns

- **Gamification over accuracy**: No streaks, hearts, points, leaderboards.
- **Grammar before context**: Never present a rule table. Show it through NPC dialogue.
- **Isolated word lists**: Vocabulary is always tied to an NPC and location.
- **Over-testing**: Chapter tests are 10 questions max. Missions are learning, not testing.
- **Perfectionism gates**: Never block progress because of a low score. Weak areas get extra missions.

## Applying this skill

When creating NPC dialogue:
- Verify vocabulary follows location sequence
- Check that new words appear in NPC speech before player is asked to produce them
- Ensure dialogue choices graduate from recognition to production

When creating missions:
- Required vocabulary matches what has been taught in this location
- Success conditions are achievable with taught vocabulary
- Failure leads to helpful NPC reaction, not dead end

When writing tutor prompts:
- NPC explains mistakes by showing correct usage in character
- NPC uses vocabulary the player has already encountered
- NPC acknowledges what the player got right first
